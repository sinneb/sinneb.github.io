first prototype of the Wakefield synth

Mon, Jan 1, 0001

The Wakefield synth: a synthesizer with a number of buildin programs that you can tweak to your heart’s desire ;). Without further ado: here’s a video showcasing the prototype:

 

The Wakefield synth uses synthcards; small cards that fit on Wakefield’s front that illustrate the current program in use. These synthcards show an abstracted graphical representation of the underlying synth program, with cutouts for the 8 led’s.

Synthcard “elephants cry” in development

Wakefield allows you to control 8 groups of 2 controllers each (totalling in 16 midi controllers). The current group is selected by using the pushbuttons and shown the corresponding LED illuminating. The two potentiometers will be “linked” to the selected group.  User can make adjustments to the selected group and the select another group and make adjustments and so on.

The underlying synthprogram is developed using pyo-patcher, a visual programming language especially developed for the Wakefield synthesize. Following screenshot shows the “elephants cry” program in pyo-patcher:

Quite different than the synthcard! The purple blocks represent the midicontrollers that are available through the Wakefield interface (101 up to 116, in blocks of two). Here a comparison of two details:

The MoogLP block is represented by a lowpass filter block in the synthcard. The Delay block is represented by the airplane (as being delayed) and the Freeverb block is represented by the cathedral. This is just an example of a synthcard. Everything is possible!

Right now I’m developing pyo-patcher somewhat further. Most recently I worked on realtime input that is trigger-based recorded in audiotables. Those tables can than be used as oscillators. Extra feature since this morning is an offset setting for oscillators in pyo…. Will keep you updated!
